- Tracking progress

Everything should reinforce learning.

# Core Principles

## Teach Through Interaction

Never default to long explanations if an interactive experience would be more effective.

Prefer:

- Simulators
- Replay tools
- Challenges
- Quizzes
- Games
- Visualizations
- Feedback loops
- Guided practice

---

## Risk Before Profit

The platform should consistently reinforce:

Capital preservation.

Proper position sizing.

Risk management.

Emotional discipline.

Process over profits.

---

## Build for Beginners

Assume users know nothing.

Avoid unnecessary jargon.

Introduce concepts gradually.

Build confidence through repetition.

---

## Encourage Critical Thinking

Do not simply tell users the answer.

Guide them toward discovering it.

Ask questions.

Provide hints.

Offer explanations after decisions are made.

# Software Engineering Standards

Always produce production-quality code.

Code should be:

- Modular
- Reusable
- Testable
- Readable
- Well documented
- Scalable

Never introduce technical debt without explaining why.

---

# Architecture Rules

Before implementing any feature:

1. Understand the request.

2. Review existing architecture.

3. Identify reusable components.

4. Consider future expansion.

5. Explain implementation.

6. Build.

7. Test.

8. Refactor.

Never skip planning.

---

# Code Quality

Prefer:

Small functions.

Small components.

Strong typing.

Meaningful names.

Composition over duplication.

Reusable hooks.

Reusable utilities.

Clear separation of concerns.

Avoid clever code.

Optimize for readability.

# Decision Framework

When multiple solutions exist:

Explain the options.

Compare trade-offs.

Recommend the best long-term approach.

Then implement it.

Never choose the fastest solution if it creates future problems.
